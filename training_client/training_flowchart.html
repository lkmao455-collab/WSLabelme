📌 协议说明：
  • 包头：固定值 0x55AA55AA，用于识别消息起始位置（大端序）
  • 长度：后续数据部分的字节数（大端序）
  • 校验和：数据部分所有字节累加和的低 32 位
  • 数据：UTF-8 编码的 JSON 字符串

4. MessageProtocol 类

calculate_checksum(data)

参数：data: bytes

返回：int (32 位校验和)

作用：计算数据的校验和，用于验证数据完整性

Python
def calculate_checksum(self, data: bytes) -> int: """计算校验和""" checksum = 0 for byte in data: checksum = (checksum + byte) & 0xFFFFFFFF # 保持 32 位 return checksum

pack(message)

参数：message: str (JSON 字符串)

返回：bytes (打包后的二进制数据)

作用：将消息打包成协议格式（包头 + 长度 + 校验和 + 数据）

Python
def pack(message: str) -> bytes: data = message.encode('utf-8') length = len(data) checksum = calculate_checksum(data) # 包头：0x55AA55AA (4 字节，大端序) header = struct.pack('>I', 0x55AA55AA) return header + struct.pack('>I', length) + struct.pack('>I', checksum) + data

receive(conn, timeout)

参数：conn: socket.socket, timeout: float (默认 30)

返回：str | None (解析后的消息)

作用：从 Socket 连接接收并解析消息

⚠️ 注意：receive() 方法会分块读取数据，确保读取完整的数据包后才返回。

6. 核心方法详解

connect()

返回：bool

作用：连接到训练服务器

Python
def connect(self) -> bool: try: self.socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM) self.socket.connect((self.host, self.port)) self.connected = True return True except Exception as e: print(f"连接失败：{e}") return False

send_request(command, data)

参数：command: str, data: Dict (可选)

返回：Dict | None

作用：发送请求并获取响应（所有方法的基础）

📌 请求结构：
JSON
{ "command": "ping", # 命令类型 "request_id": "1234567890_1", # 唯一请求 ID "timestamp": 1234567890.123, # 时间戳 "task_id": "xxx" # 其他参数（可选） }

create_task(params)

参数：params: Dict

返回：task_id (str) | None

作用：创建新的训练任务，返回任务 ID

start_training(task_id)

参数：task_id: str

返回：bool

作用：启动指定的训练任务

get_task_status(task_id)

参数：task_id: str

返回：Dict | None

作用：获取任务状态（status, progress 等）

get_progress(task_id)

参数：task_id: str

返回：Dict | None

作用：获取训练进度详情（epoch, loss, accuracy 等）

monitor_training(task_id, poll_interval)

参数：task_id: str, poll_interval: float (默认 2.0)

作用：持续监控训练进度直到完成

7. create_task 参数详解

参数 类型 默认值 作用 建议
model_type str "detect" 模型类型（detect/segment/classify） 根据任务选择
image_size int 640 输入图像尺寸（像素） 320-1280
dataset str "data" 训练数据集路径 确保路径存在
epochs int 50 训练轮次数 30-100
batch_size int 32 每批次样本数 16/32/64
learning_rate float 0.001 学习率 0.001-0.01
trainset_ratio float 0.9 训练集占比 0.8-0.95
⚠️ 参数影响：
  • epochs 太少：模型可能未充分学习（欠拟合）
  • epochs 太多：可能过拟合，训练时间增长
  • batch_size 太小：训练不稳定，但内存占用少
  • learning_rate 太大：可能不收敛或震荡

8. 完整使用示例

Python
from training_client import TrainingClient import time # ========== 1. 创建客户端 ========== client = TrainingClient(host="127.0.0.1", port=8888) # ========== 2. 连接服务器 ========== if not client.connect(): print("❌ 连接失败") exit() # ========== 3. 验证连接 ========== if not client.ping(): print("❌ 服务器无响应") client.disconnect() exit() # ========== 4. 创建训练任务 ========== params = { "model_type": "detect", "image_size": 640, "dataset": "data/my_dataset", "epochs": 50, "batch_size": 32, "learning_rate": 0.001, "trainset_ratio": 0.9 } task_id = client.create_task(params) if not task_id: print("❌ 创建任务失败") client.disconnect() exit() print(f"✅ 任务创建成功：{task_id}") # ========== 5. 启动训练 ========== if client.start_training(task_id): print("✅ 训练已启动") # ========== 6. 监控进度 ========== print("开始监控训练进度...") try: while True: status = client.get_task_status(task_id) progress = client.get_progress(task_id) if progress and "progress" in progress: p = progress["progress"] epoch = p.get("epoch", 0) total = p.get("total_epochs", 1) loss = p.get("loss", 0) accuracy = p.get("accuracy", 0) print(f"轮次 {epoch}/{total} | 损失：{loss:.4f} | 准确率：{accuracy:.4f}") # 检查是否完成 if epoch >= total: print("✅ 训练完成!") break time.sleep(2) # 每 2 秒查询一次 except KeyboardInterrupt: print("\n⚠️ 监控被用户中断") # ========== 7. 断开连接 ========== client.disconnect() print("✅ 已断开连接")

10. 消息传递原理

消息流程
┌─────────────────────────────────────────────────────────────────┐ │ 客户端 → 服务器 请求 │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ Python 字典 → json.dumps() → JSON 字符串 │ │ ↓ MessageProtocol.pack() │ │ [Header:0x55AA55AA][Length][Checksum][Data] │ │ ↓ socket.sendall() │ │ TCP 网络传输 │ │ │ └─────────────────────────────────────────────────────────────────┘ ┌─────────────────────────────────────────────────────────────────┐ │ 服务器 → 客户端 响应 │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ TCP 网络接收 → 解析包头 → 验证 Checksum → 读取 Data │ │ ↓ json.loads() │ │ Python 字典 {"code": 100, "data": {...}} │ │ │ └─────────────────────────────────────────────────────────────────┘
✅ 关键点：
  • 所有通信都是基于 TCP 的同步请求 - 响应模式
  • 每个请求都有唯一的 request_id（时间戳 + 计数器）
  • 包头用于识别消息边界，校验和用于验证数据完整性
  • 超时处理：connect 默认超时，receive 可设置 timeout
