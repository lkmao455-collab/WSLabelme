# 防止多次启动功能说明

## 功能概述

Labelme 现在支持防止多次启动的功能，使用共享内存技术实现进程间通信，确保同一时间只能运行一个 Labelme 实例。

## 功能特性

### 核心特性
- **共享内存锁**: 使用 Python 的 `multiprocessing.shared_memory` 模块实现进程间通信
- **自动清理**: 程序退出时自动清理共享内存，防止资源泄漏
- **僵尸进程检测**: 自动检测和清理僵尸进程的共享内存
- **跨平台支持**: 支持 Windows、Linux、macOS 等主流操作系统
- **优雅降级**: 当共享内存不可用时，允许程序正常启动

### 用户体验
- **友好提示**: 当检测到已有实例运行时，显示清晰的警告消息
- **自动退出**: 阻止重复启动，避免资源冲突
- **日志记录**: 详细的日志记录，便于问题排查

## 技术实现

### 核心函数

#### `check_single_instance()`
检查是否已有 Labelme 实例在运行。

**工作流程:**
1. 尝试创建共享内存块（名称格式：`labelme_instance_lock_{PID}`）
2. 如果创建成功，写入当前进程ID并返回 `True`
3. 如果创建失败（共享内存已存在），读取存储的进程ID
4. 检查该进程是否仍在运行
5. 如果进程已结束，清理旧的共享内存并重新尝试
6. 如果进程仍在运行，返回 `False`

#### `_is_process_running(pid)`
检查指定PID的进程是否在运行。

**跨平台实现:**
- **Windows**: 使用 `psutil.pid_exists(pid)` 检查
- **Unix/Linux/macOS**: 使用 `os.kill(pid, 0)` 检查

### 集成点

在 `labelme/__main__.py` 的 `main()` 函数开始处添加了实例检查：

```python
def main():
    # 检查是否已有实例在运行
    if not check_single_instance():
        logger.error("Labelme instance is already running!")
        # 显示错误消息给用户
        QtWidgets.QApplication(sys.argv)  # 创建临时应用来显示消息框
        msg_box = QtWidgets.QMessageBox()
        msg_box.setIcon(QtWidgets.QMessageBox.Warning)
        msg_box.setWindowTitle("Labelme Already Running")
        msg_box.setText("Labelme is already running!")
        msg_box.setInformativeText("Only one instance of Labelme can run at a time.")
        msg_box.setStandardButtons(QtWidgets.QMessageBox.Ok)
        msg_box.exec_()
        sys.exit(1)
    
    # ... 其余启动代码
```

## 使用方法

### 正常启动
```bash
python -m labelme
```

### 重复启动检测
当尝试启动第二个实例时：
1. 程序会检测到已有实例在运行
2. 显示警告对话框："Labelme is already running!"
3. 程序自动退出，返回错误码 1

### 关闭实例后重新启动
1. 关闭第一个 Labelme 实例
2. 共享内存会自动清理
3. 可以正常启动新的实例

## 故障排除

### 常见问题

#### 1. 共享内存不可用
**症状**: 程序正常启动，但不进行实例检查
**原因**: 系统不支持 `multiprocessing.shared_memory`（如 Windows 10 之前的系统）
**解决方案**: 功能会优雅降级，允许启动多个实例

#### 2. 僵尸进程残留
**症状**: 程序显示"已有实例运行"但实际没有
**原因**: Labelme 异常退出，共享内存未清理
**解决方案**: 系统会自动检测并清理僵尸进程的共享内存

#### 3. 权限问题
**症状**: 无法创建共享内存
**原因**: 操作系统权限限制
**解决方案**: 功能会优雅降级，允许启动多个实例

## 技术细节

### 共享内存设计
- **名称格式**: `labelme_instance_lock_{PID}`
- **大小**: 4字节（存储进程ID）
- **数据格式**: 32位整数（进程ID）
- **生命周期**: 程序启动时创建，退出时自动清理

### 进程检测机制
- **实时检测**: 每次启动时都检查进程状态
- **僵尸清理**: 自动清理已结束进程的共享内存
- **错误处理**: 各种异常情况的优雅处理

### 兼容性考虑
- **Python 版本**: 需要 Python 3.8+（`shared_memory` 模块要求）
- **操作系统**: Windows 10+、Linux、macOS
- **依赖**: 可选依赖 `psutil`（Windows 进程检测）
