【任务背景】
基于 Labelme 进行二次开发，在现有 GUI 界面中新增一个 Dock 面板，实现通过 SSH 向远端 Linux 设备传输模型文件，并扩展为支持多设备管理与自动部署的工具。

---

# 一、基础功能（原有功能）

## 1. Dock 面板

新增一个可停靠（Dockable）的侧边面板，用于文件传输与设备管理。

---

## 2. SSH 连接配置区域

包含以下组件：

* IP 地址输入框
* 端口输入框（默认 22）
* “连接测试”按钮

功能要求：

* 点击“连接测试”后，通过 SSH 尝试连接远端设备
* 成功：按钮显示绿色状态（或文本变为“连接成功”）
* 失败：按钮显示红色状态（或文本变为“连接失败”）
* 必须设置超时机制（3~5 秒）
* 必须在独立线程中执行

---

## 3. 设备类型选择（组合框）

设备类型列表：

ERV / GSV / BRV / AFRV / CRV / GSV2 / AFSV / GHV

自动匹配登录凭据：

* ERV / GSV2 → root / root
* GSV / CRV / GHV → root / SenvisionTech
* AFRV / AFSV → root / 1

要求：

* 切换设备类型自动更新用户名和密码
* 用户名密码可隐藏或只读显示

---

## 4. 文件选择

* “选择文件”按钮
* 文件路径显示标签

---

## 5. 文件传输

* 使用 SSH（paramiko）+ SFTP
* 目标路径固定：/mmcblk1p2
* 支持覆盖或提示

---

## 6. 进度条

* 显示实时传输进度（百分比）
* 可选：显示速度 / 剩余时间

---

## 7. 多线程

以下必须使用独立线程：

* SSH连接测试
* 文件传输
* 状态检测

UI 更新必须通过主线程（Signal/Slot）

---

# 二、高级功能扩展（重点）

---

## 8. 断点续传（Resumable Upload）

功能要求：

* 支持文件上传中断后继续传输

* 实现方式：

  * 上传前检查远端文件是否存在
  * 获取远端文件大小（SFTP stat）
  * 从断点位置继续写入（seek + append）

* UI要求：

  * 显示“断点续传中”状态
  * 若文件已完整存在，可提示“已存在，无需上传”

* 异常处理：

  * 网络中断自动暂停
  * 支持用户手动重试

---

## 9. 多设备管理

新增“设备列表管理模块”，支持：

* 添加设备（IP + 类型）
* 删除设备
* 编辑设备
* 设备列表展示（QList / QTable）

功能要求：

* 支持保存设备列表（本地 JSON 文件）
* 支持选择设备后自动填充连接信息
* 支持批量操作（可选增强）：

  * 批量连接测试
  * 批量文件传输

---

## 10. 日志面板（Log Panel）

新增一个日志显示区域（可在 Dock 或单独 Tab）：

日志内容包括：

* SSH连接状态
* 传输进度
* 错误信息
* 操作记录

要求：

* 按时间顺序显示
* 不同级别日志（INFO / WARNING / ERROR）
* 不同颜色区分（可选）
* 支持自动滚动到底部
* 支持日志导出（txt 文件，可选增强）

---

## 11. 一键部署（核心增强功能）

新增“一键部署”按钮，实现：

功能流程：

1. 自动连接目标设备
2. 上传模型文件
3. （可选）执行远端命令，例如：

   * 解压文件
   * 移动文件
   * 重启服务

示例命令：

```bash
chmod +x /mmcblk1p2/xxx
./deploy.sh
```

实现要求：

* 使用 SSH exec_command 执行远端命令
* 显示执行输出（stdout / stderr）
* 支持失败回滚或提示

UI要求：

* 显示部署状态（进行中 / 成功 / 失败）
* 显示步骤进度（连接 → 上传 → 执行）

---

# 三、技术要求

推荐技术栈：

* GUI：PyQt5 / PySide2（与 Labelme 一致）
* SSH：paramiko
* 多线程：QThread 或 threading
* 数据存储：JSON（设备列表）

---

# 四、代码结构要求

必须模块化设计：

* ui/ → 界面
* ssh/ → SSH连接与传输
* worker/ → 线程处理
* manager/ → 设备管理
* log/ → 日志系统

---

必须处理：

* SSH连接失败
* 认证失败
* 文件不存在
* 网络中断
* 权限问题
